## 📝타입스크립트 공식 문서 The Basics 파트의 번역

[Documentation - The Basics](https://www.typescriptlang.org/docs/handbook/2/basic-types.html)

### 개요

- 자바스크립트의 모든 값에는 다양한 연산을 실행할 때 관찰할 수 있는 일련의 동작이존재
- 추상적으로 들리겠지만, 간단한 예로 `message`라는 변수에 대해 실행할 수 있는 몇 가지 연산을 생각

**코드분석**

- 실행 가능한 첫 번째 코드 줄은 `toLowerCase`라는 속성에 액세스한 다음 이를 호출
- 두 번째는 메시지를 직접 호출하려고 시도

```jsx
// Accessing the property 'toLowerCase'
// on 'message' and then calling it
message.toLowerCase();
// Calling 'message'
message();
```

**문제점은?**

- 우리가 메시지의 가치를 모른다고 가정하면(이것은 매우 일반적인 일)
- 이 코드를 실행하려고 하면 어떤 결과를 얻게 될지 확실하게 말할 수 ❌
- 각 작업의 동작은 전적으로 우리가 처음에 가졌던 가치에 따라 달라 짐

**메시지가 다음과 같이 정의되었다고 가정**

- `message.toLowerCase()`를 실행하려고 하면 동일한 문자열이 소문자로만 표시
  - 두 번째 코드 줄은 어떻습니까? JavaScript에 익숙하다면 예외가 발생하여 실패(= message는 변수이지 함수가 아니기때문에)

```jsx
const message = "Hello World!";
```

🌅 **이런 실수는 피할 수 있으면 좋을 것 같다**

</aside>

**자바스크립트 런타임**

- 코드를 실행할 때 자바스크립트 런타임이 수행할 작업을 선택하는 방식은 값의 유형, 즉 어떤 종류의 동작과 기능을 가지고 있는지 파악
- `"Hello World!"`라는 문자열을 함수로 호출할 수 없다는 것이 `TypeError`가 암시하는 바의 일부
- 문자열 및 숫자와 같은 일부 값의 경우 런타임에 `typeof` 연산자를 사용하여 해당 유형을 식별할 수 있다
- 그러나 함수와 같은 다른 값의 경우 해당 유형을 식별할 수 있는 런타임 메커니즘이 ❌

**아래 함수는?**

- 코드를 읽으면 아래 함수가 호출 가능한 플립 프로퍼티가 있는 객체가 주어진 경우에만 작동한다는 것을 알 수 있다,
- **그러나**, 자바스크립트는 코드가 실행되는 동안 이 정보를 확인할 수 있는 방식으로 표시 ❌
- 순수 자바스크립트에서 `fn`이 특정 값으로 무엇을 하는지 알 수 있는 유일한 방법은 함수를 호출하여 어떤 일이 일어나는지 보는 것
- 이러한 동작은 코드가 실행되기 전에 코드가 어떤 동작을 할지 예측하기 어렵게 만들며, 이는 코드를 작성하는 동안 코드가 어떤 동작을 할지 알기 어렵다는 것을 의미

```jsx
function fn(x) {
  return x.flip();
}
```

### 정적 유형 검사(Static type-checking)

**이것이 타입스크립트를 쓰는 이유**

- TypeScript와 같은 타입 검사기는 이러한 정보를 사용하여 언제 문제가 발생할 수 있는지 알려 줄 수 있다

**ts로 사용시 message변수를 함수가 아닌데 호출하려할 시**

- TypeScript를 사용하여 마지막 샘플을 실행하면 처음에 코드를 실행하기 전에 오류 메시지가 표시

```jsx
const message = "hello!";

message();
**This expression is not callable.
  Type 'String' has no call signatures.**
```

### 예외가 아닌 실패

**자바스크립트의 유연성**

- 호출할 수 없는 것을 호출하려고 하면 오류가 발생해야 한다고 명시
- 객체에 존재하지 않는 프로퍼티에 액세스하면 에러가 발생해야 한다고 생각
- **그러나** 자바스크립트는 **다른 동작을 제공하고 정의되지 않은 값을 반환**합니다:

**바닐라 자바스크립트에서 정의되지 않은 프로퍼티를 가진 속성에 접근**

- `error`가 나타나지않고 `undefined`를 반환한다

```jsx
const user = {
  name: "Daniel",
  age: 26,
};
**user.location; // returns undefined**
```

**타입스크립트에서 정의되지 않은 프로퍼티를 가진 속성에 대한 접근**

- 타입스크립트에선 위치가 정의되지 않았다는 오류를 발생
  - `**'{ 이름: 문자열;' 유형에 '위치' 속성이 없습니다. 나이: 숫자; }'.**`

```jsx
const user = {
  name: "Daniel",
  age: 26,
};

**user.location;**
**Property 'location' does not exist on type '{ name: string; age: number; }'.**
```
